コマンド

init
リポジトリを初期化する
clone
リポジトリをクローンする
add
現在のディレクトリ(または、変更対象ファイル)の内容を保存し、コミット対象に加える
rm
指定したファイルを削除する
commit
インデックスの内容をコミットとして保存する
merge
分岐された内容を分岐元にマージしてコミットする
fetch
ほかのリポジトリから変更を取得する
pull
ほかのリポジトリの変更点をローカルリポジトリにマージする
push
公開リポジトリに自分のリポジトリの内容を送信する
log
コミット履歴を表示する
diff
ある時点からの変更内容をパッチ形式の差分で表示する
status
変更されたファイルを表示する
blame
指定したファイルの各行がどのコミットで変更されたかを表示する

単語

branch
軽度の開発やバグフィックスなどの作業はbranchを使用して開発する。
commit object
特定リビジョンの情報を含んだオブジェクトのこと
head
ブランチの先端にあるコミットへの名前のついた参照 のこと
HEAD
現在のブランチ
repository
参照(refs)と参照から到達可能な全オブジェクトを 含んだオブジェクトＤＢの集合
revision
オブジェクトＤＢ 内に格納されたファイルとディレクトリ の特定の状態
tag
タグ(tag) や コミットオブジェクト を 指し示す 参照(ref) のこと
tag object
他のオブジェクトを指し示す 参照(ref) を含んでいる オブジェクト のこと
topic branch
各開発ラインを識別する為に開発者が使用する通常の git ブランチ
tree
依存している blob と ツリーオブジェクトも含めた 作業ツリー または ツリーオブジェクト全体
tracking branch
他の リポジトリ からの変更を追跡する為に使用される 通常の git ブランチ
tree object
blob または ツリーオブジェクトへの参照と、ファイル名とモードの一覧を含んだ オブジェクト
working tree
実際にチェックアウトされたファイルのツリー
workspace
ソースコードなどを編集するための領域
